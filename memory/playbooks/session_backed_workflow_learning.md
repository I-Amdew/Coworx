# Session-Backed Workflow Learning Playbook

## Purpose
Let Coworx get smarter from real browser/app use without learning credentials.

## Steps
1. User approves target and credential-safe access route.
2. Coworx maps the approved workflow through a leased browser/API/connector lane or Computer Use target lock.
3. Coworx stores private observations in ignored paths.
4. Coworx identifies reusable generic steps.
5. Coworx sanitizes private data.
6. Coworx updates framework playbooks/templates with sanitized lessons.
7. Coworx adds or updates a smoke/regression test.

## What To Learn
- page routes;
- tab names;
- selectors;
- workflow order;
- common stop conditions;
- draft formats;
- review checklists;
- user preferences that are safe to store privately.

## What Not To Learn
- credential values;
- cookies;
- tokens;
- MFA answers;
- recovery codes;
- payment data;
- security answers;
- private message/calendar/meeting contents in shippable memory.
