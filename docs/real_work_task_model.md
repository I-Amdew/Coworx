# Real Work Task Model

Coworx handles real work by splitting execution into safe phases.

## Phase 1: Understand
Read the request, classify risk, identify tools, and define success criteria.

## Phase 2: Map
Use installed plugins, Browser Use, Playwright, or Computer Use to map the approved app, page, file, or workflow.

## Phase 3: Draft Or Build
Create the work product:
- meeting summary;
- action-item list;
- document;
- spreadsheet;
- slide deck;
- message draft;
- calendar draft;
- organized files;
- report.

## Phase 4: Review
Run reviewer and safety checks. Confirm the output matches the task and no private memory is being committed.

## Phase 5: Act
Complete safe local actions automatically when permitted. Pause for user approval before external communication, scheduling, submission, publishing, deletion, payment, settings, or sensitive-data transmission.

## Phase 6: Learn
Write safe workflow memory. Keep private user-specific memory in ignored paths.

## Examples
- Meeting to slides: transcribe or read notes, summarize, create outline, build deck, review, save locally.
- Dashboard summary: map approved dashboard, read metrics, produce report, update workflow memory.
- Message drafting: read approved context, draft message, stop for approval before sending.
- Meeting scheduling: draft event details, stop for approval before creating or inviting.
- File organization: propose moves, perform local moves only when approved, log changes.
